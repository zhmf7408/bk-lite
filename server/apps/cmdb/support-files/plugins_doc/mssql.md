### 说明
该插件通过 MSSQL 原生协议直连实例，读取版本与关键配置项，并标准化同步至 CMDB。


### 操作入口与执行位置
你需要在 CMDB Web 页面完成配置：
1. 进入“CMDB → 管理 → 自动发现 → 采集 → 专业采集”。
2. 在插件卡片中选择 **MSSQL**。
3. 点击“新增任务”。

说明：任务实际执行发生在你选择的“接入点”上；文档中的连通性自测命令，均应在接入点机器上执行。


### 版本兼容性
- 适用于支持 TDS/ODBC 连接的 Microsoft SQL Server 实例。

### 前置要求
在开始之前，请先确认以下条件：

1. **目标信息已明确**
   - MSSQL 地址：IP 或域名。
   - 端口：默认 `1433`，如有改动以实际为准。
   - 数据库名：建议填写可正常登录并允许查询系统视图的数据库，默认可使用 `master`。
2. **网络连通已放通**
   - 接入点 → MSSQL 的 `TCP/<port>` 可达（安全组/防火墙/路由均放通）。
3. **已准备采集专用账号（只读/查询为主）**
   - 不要复用业务账号或管理员账号。
   - 建议按最小权限创建采集账号，至少保证能够连接实例并查询版本、配置、服务账户等信息。
4. **接入点运行环境满足驱动要求**
   - Stargazer 插件当前依赖 ODBC 方式连接 SQL Server，请确保接入点环境具备可用的 SQL Server ODBC 驱动。


### 凭据字段说明
- 凭据字段：用户名（`user`）、密码（`password`）、端口（`port`）、数据库名（`database`）。
- `user`：用于登录目标 MSSQL 的账号名称。建议使用单独创建的采集账号。
- `password`：上述账号对应的密码。
- `port`：目标 MSSQL 实例对外提供服务的端口号，默认 `1433`。
- `database`：采集时连接使用的数据库名，推荐填写 `master` 或具备系统视图访问能力的业务数据库。


### 采集内容
| Key 名称            | 含义       |
| :------------------ | :--------- |
| mssql.ip_addr       | 实例 IP    |
| mssql.port          | 监听端口   |
| mssql.version       | MSSQL 版本 |
| mssql.db_name       | 数据库名称 |
| mssql.max_conn      | 最大连接数 |
| mssql.max_mem       | 最大内存   |
| mssql.order_rule    | 排序规则   |
| mssql.fill_factor   | 填充因子   |
| mssql.boot_account  | 启动账户   |

> 补充说明：当前 CMDB MSSQL 采集实现基于协议采集链路，字段名称遵循 `model_config.xlsx` 中的 `attr-mssql` 定义；旧版 Windows 本地注册表发现逻辑未在本次实现中直接迁移。
