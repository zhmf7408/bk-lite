# InfluxDB 告警说明

## 文档说明

本文档说明 InfluxDB 监控默认提供的告警模板。这些告警主要覆盖以下常见运维风险：

- 序列规模过快膨胀，导致高基数风险
- HTTP 认证异常、4XX/5XX 接口访问异常
- 写入链路出现丢点或持久化失败
- 运行时堆内存持续膨胀

这些告警用于帮助使用者在 InfluxDB 出现异常征兆时尽早介入，避免问题进一步影响写入、查询和实例稳定性。

## 使用建议

- 当前阈值为通用默认值，适合作为 InfluxDB 接入后的初始告警基线。
- 如果某实例承载大量高基数时序数据，建议重点关注“序列数过高”和“堆内存占用过高”两类告警。
- 如果某实例承担核心写入链路，建议重点关注“写入接口丢点过高”和“写入接口持久化失败过高”。

## 告警模板

| 告警名称 | 关联指标 | 检测算法 | 触发阈值 | 指标含义 | 告警详细说明 | 处置建议 |
| --- | --- | --- | --- | --- | --- | --- |
| 数据库序列数过高 | `influxdb_database_numSeries` | `max` | `critical: >= 5000000` `error: >= 1000000` `warning: >= 500000` | 单个数据库的序列总数 | 序列数过高会带来明显的高基数压力，通常会推动内存占用上升、索引压力增大，并最终影响查询性能和实例稳定性。 | 优先检查标签设计是否过细、是否存在高基数维度，必要时调整写入模型或拆分数据。 |
| HTTP认证失败次数过高 | `influxdb_httpd_authFail_rate` | `max` | `critical: >= 30` `warning: >= 10` | HTTP 接口认证失败次数 | HTTP 认证失败次数突增通常意味着访问配置错误、凭据失效，或存在异常尝试访问接口的行为。 | 检查上游调用配置、认证凭据、认证策略以及异常访问来源。 |
| HTTP 4XX 次数过高 | `influxdb_httpd_clientError_rate` | `max` | `critical: >= 100` `warning: >= 20` | HTTP 4XX 响应次数 | HTTP 4XX 持续升高通常意味着请求参数错误、接口调用方式异常或上游接入逻辑存在问题。 | 检查上游请求参数、接口访问路径、认证信息和调用方发布变更。 |
| HTTP 5XX 次数过高 | `influxdb_httpd_serverError_rate` | `max` | `critical: >= 20` `warning: >= 5` | HTTP 5XX 响应次数 | HTTP 5XX 持续升高通常意味着实例内部处理异常、资源压力上升，或服务端写入/查询链路发生故障。 | 检查 InfluxDB 实例日志、资源使用情况、磁盘状态及最近变更。 |
| 写入接口丢点过高 | `influxdb_httpd_pointsWrittenDropped_rate` | `max` | `critical: >= 100` `warning: >= 10` | 写入接口丢失的数据点数量 | 表示写入链路已接收数据点但未能继续处理，持续升高说明写入路径可能存在资源不足、限流或内部异常。 | 检查写入量是否突增、接口错误日志、实例资源使用情况以及存储路径健康度。 |
| 写入接口持久化失败过高 | `influxdb_httpd_pointsWrittenFail_rate` | `max` | `critical: >= 100` `warning: >= 10` | 已接收但未成功持久化的数据点数量 | 表示写入请求已经进入服务端，但最终未能成功落盘。该问题通常比丢点更严重，往往意味着磁盘、存储引擎或内部写入链路已经出现异常。 | 优先检查磁盘空间、磁盘性能、实例日志和存储引擎状态，必要时暂停高峰写入并排查底层故障。 |
| 堆内存占用过高 | `influxdb_runtime_HeapAlloc` | `last_over_time` | `critical: >= 4294967296` `warning: >= 2147483648` | Go runtime 当前堆内存占用 | 运行时堆内存持续升高通常意味着高基数、写入压力、查询压力或内部对象堆积已经推动进程进入高内存风险区。 | 结合序列数、写请求速率和查询请求速率一起分析，必要时优化数据模型、拆分实例或扩容资源。 |

## 告警判定方式说明

- `max` 适用于短时间内一旦升高就值得关注的异常事件类指标，例如认证失败、丢点和持久化失败。
- `last_over_time` 适用于状态或资源占用类指标，强调“当前是否仍处于高风险状态”。
- `warning` 阈值用于提前介入，`error` 或 `critical` 阈值通常表示已经进入明显影响稳定性的风险区间。

## 重点告警建议优先启用

如果需要先启用一组最关键的 InfluxDB 告警，建议优先启用以下模板：

- `数据库序列数过高`
- `写入接口丢点过高`
- `写入接口持久化失败过高`
- `堆内存占用过高`

## 使用建议

- 如果实例承载核心写入链路，建议优先启用写入异常相关告警。
- 如果实例存在高基数风险，建议优先启用“数据库序列数过高”和“堆内存占用过高”告警。
- 如果业务访问来源较多，建议保留“HTTP认证失败次数过高”告警，用于及早发现接入配置错误或异常访问。
- 如果近期有接口改造或网关变更，建议同时启用“HTTP 4XX 次数过高”和“HTTP 5XX 次数过高”告警。
