# 运营分析支持仪表盘统一筛选

日期：2026-04-12

## 背景

当前运营分析的仪表盘已支持组件化画布、数据源绑定、组件级参数配置以及全局时间范围控制，但除时间范围外，缺少能够在仪表盘层面对多个组件统一生效的筛选能力。

现状下，用户若希望在同一个仪表盘中按业务、环境、实例、项目、责任人等维度切换观察口径，需要逐个组件修改查询参数，操作成本高，且容易因组件参数不一致导致结果无法横向对比。

同时，不同数据源对同一业务语义往往使用不同参数名或不同参数结构表达，若直接要求用户在画布上理解每个组件的数据源细节，会显著提高配置门槛，也不利于模板化复用。

因此，运营分析需要为仪表盘类型画布提供统一筛选能力：由画布定义可复用的统一筛选项，由组件声明哪些筛选对其生效，并由数据源提供清晰的参数归类与可控边界，使用户能够在一个画布内统一切换分析口径，同时保持组件查询结果可预期、可复用、可迁移。

## 需求描述

运营分析的仪表盘类型画布新增“统一筛选”能力。统一筛选由画布集中定义，可在画布顶部或固定筛选区域统一操作，并控制当前画布内已绑定该筛选项的组件查询结果。

统一筛选能力需覆盖筛选项定义、组件绑定、生效规则、数据源参数设计、保存回显、模板复用以及 YAML 导入导出一致性。

本期统一筛选采用“同 key 公共筛选”原则：仪表盘中可配置的统一筛选项，仅允许从组件所绑定数据源中标记为“参数”的字段中选择；当多个组件的数据源参数 key 相同，且组件显式启用该 key 的统一筛选后，这些组件可被同一个公共筛选项同时控制。

## 术语说明

- 参数：指数据源中对用户可见、可由用户输入或选择的查询字段，用于参与组件查询条件拼装。
- 固定参数：指数据源内部预设且不允许用户在仪表盘或组件中修改的参数。
- 组件私有参数：指仅对当前组件生效、不会提升为仪表盘公共筛选的参数。
- 公共筛选 key：指仪表盘统一筛选识别组件共用筛选项的唯一键。
- 同 key 公共筛选：指多个组件只有在其数据源参数 key 完全相同，且该参数属于可参与统一筛选的“参数”字段时，才可共用同一个仪表盘筛选项。
- 可参与统一筛选字段：指数据源中被标记为“参数”，且允许出现在仪表盘统一筛选配置中的字段；固定参数及其他内部字段不属于该范围。

### 1）统一筛选对象与适用范围

- 统一筛选能力本期仅适用于仪表盘类型画布。
- 仪表盘可配置一组画布级统一筛选项。
- 统一筛选项用于表达画布层的通用分析维度，如时间范围、命名空间、业务、环境、实例、项目等。
- 用户在仪表盘中调整统一筛选值后，当前画布内所有已绑定该筛选项的组件需按最新条件重新查询。
- 未绑定该筛选项的组件不应受影响。
- 不存在数据源绑定关系或不支持对应筛选的组件，不应被统一筛选强行控制。
- 统一筛选能力不要求所有组件必须共享同一组筛选项，画布可仅对部分组件生效。

### 2）画布级筛选定义规则

- 仪表盘支持定义多个统一筛选项。
- 仪表盘可配置为统一筛选项的字段，仅允许来自数据源中标记为“参数”的字段，不允许直接选择固定参数或其他不对用户开放的内部字段。
- 每个统一筛选项需具备稳定标识、显示名称、值类型、展示控件、默认值及必要的展示说明。
- 统一筛选项至少支持以下输入形态：单选下拉、多选下拉、关键字输入、时间范围。
- 统一筛选项需支持显式配置默认值；未配置默认值时，系统按空值处理。
- 仪表盘保存后，统一筛选定义与默认值需一并保存。
- 用户再次打开仪表盘时，系统应按保存结果回显统一筛选定义与默认值。
- 统一筛选项的显示顺序应可配置并保持稳定，避免同一画布重复打开后顺序变化。
- 统一筛选项需支持启用与停用；停用后不再对组件生效，但不要求删除其定义。
- 本期不要求统一筛选提供复杂条件编排、嵌套逻辑组合或筛选方案收藏能力。

### 3）组件绑定规则

- 统一筛选是否对某个组件生效，不由画布全局强制决定，而由组件配置明确声明。
- 组件需支持为每个统一筛选项配置是否绑定。
- 当组件绑定统一筛选项后，系统应明确该筛选项映射到当前组件所使用数据源中的哪个查询参数。
- 组件参与公共筛选的前提，是其数据源中存在可用于统一筛选的“参数”字段；不属于“参数”的字段不得参与公共筛选。
- 同一个统一筛选项应允许在不同组件中映射到不同的数据源参数名，以适配不同数据源的参数差异。
- 当多个组件使用相同参数 key，且这些参数均属于可参与统一筛选的“参数”字段时，系统应允许用户基于该 key 创建一个公共筛选项，并同时作用于这些组件。
- 当多个组件参数名称不同，即使语义接近，系统本期也不要求自动合并为同一个公共筛选项。
- 当组件未配置某个统一筛选项的绑定关系时，即使画布存在该筛选项，也不应影响该组件查询。
- 当组件绑定关系失效，例如数据源参数不存在、参数类型不兼容或组件更换了数据源，系统需给出明确提示。
- 组件复制、仪表盘复制后，统一筛选绑定关系需随组件配置一并保留。
- 组件删除后，其绑定关系自动失效，不要求保留孤立绑定配置。

### 4）统一筛选生效规则

- 用户调整统一筛选值后，所有命中绑定关系且当前可正常查询的组件需按最新条件刷新。
- 刷新应以组件重新发起正式查询为准，不得仅在前端对已有结果做表面过滤后冒充统一筛选生效。
- 统一筛选与组件已有参数共存时，系统需保证结果可预期，不允许同一个参数在不同来源下出现无法判断的冲突行为。
- 当统一筛选项无值且组件允许空值时，系统按未传该筛选处理。
- 当统一筛选项无值且组件不允许空值时，系统需明确提示该组件当前缺少必要筛选条件。
- 统一筛选项对组件参数的影响范围，仅限于组件已声明绑定的参数，不得扩大到组件其他未绑定参数。
- 统一筛选项生效后，组件级私有参数仍应保持可配置，但其生效范围仅限于当前组件。
- 本期不要求支持组件之间的点击联动、钻取联动、图表选区联动等事件型筛选能力。

### 5）参数优先级与冲突规则

- 系统需明确区分数据源固定参数、画布统一筛选参数、组件私有参数和数据源默认参数。
- 当同一参数同时存在多处配置来源时，系统应按统一、稳定的优先级生效，避免不同组件行为不一致。
- 参数优先级应至少满足以下顺序：固定参数优先于统一筛选参数，统一筛选参数优先于组件私有参数，组件私有参数优先于数据源默认参数。
- 数据源定义为固定参数的内容，不应被画布统一筛选或组件私有参数覆盖。
- 被组件声明为受统一筛选控制的参数，在统一筛选有值时，应优先使用统一筛选值。
- 组件私有参数仅用于当前组件自身查询，不应反向改写画布统一筛选值。
- 数据源默认参数仅在没有更高优先级配置时作为兜底值使用。
- 当统一筛选值与组件私有参数在语义上冲突时，系统需采用明确、稳定的单一结果，不允许同一组件一次查询同时携带两套冲突值。

### 6）数据源参数设计规则

- 数据源需对查询参数进行明确分类，以支持画布统一筛选与组件私有参数协同工作。
- 数据源参数至少需区分以下语义：固定参数、可被统一筛选接管的参数、仅组件私有使用的参数。
- 本期可被仪表盘统一筛选选择的字段范围，仅限于数据源中标记为“参数”的字段。
- 数据源参数 key 在同一数据源内需保持稳定；当参数 key 变更时，系统需将其视为公共筛选绑定关系变化，而非无感知兼容更新。
- 数据源参数除保留参数名、参数类型、默认值外，还需明确该参数可否被统一筛选接管。
- 对于允许被统一筛选接管的参数，数据源需能够表达其业务语义，而不仅是技术参数名，以支持不同数据源间的统一筛选映射。
- 对于枚举类筛选参数，数据源需能够表达可选项来源；若本期不支持动态来源，则至少支持稳定、明确的静态可选项定义。
- 对于时间范围类参数，数据源需明确其时间语义与格式要求，避免不同数据源使用统一筛选后返回口径不一致。
- 对于不允许用户调整的内部参数，数据源需将其标识为固定参数，避免在画布或组件配置时被误暴露。
- 数据源切换、编辑或下线后，系统需能够识别哪些统一筛选绑定关系受到影响，并给出明确提示。

### 7）数据源与统一筛选的映射规则

- 本期公共筛选的合并依据为参数 key 是否相同。
- 当多个组件引用的数据源参数 key 相同，且这些参数均属于可参与统一筛选的“参数”字段时，用户应能够通过一个统一筛选项同时控制这些组件。
- 当多个组件引用不同数据源，且参数 key 不同，即使它们表达相近业务语义，系统本期也不要求自动视为同一个公共筛选项。
- 当数据源参数语义不兼容时，系统不应强制建立统一筛选映射。
- 当同一个组件绑定多个统一筛选项时，各筛选项应分别作用于其声明的参数，不得互相覆盖或隐式串改。
- 当统一筛选项需要映射到复杂参数结构时，本期仅要求支持明确、稳定的单层参数映射；不要求支持任意深层嵌套结构编排。

### 8）画布保存、回显与复用规则

- 仪表盘保存时，统一筛选定义、默认值、组件绑定关系应作为画布配置的一部分一并保存。
- 用户再次进入仪表盘时，系统需按保存结果完整回显统一筛选项和组件绑定关系。
- 仪表盘复制、模板复用、跨环境迁移时，统一筛选定义与绑定关系需一并保留。
- 当目标环境中数据源发生变化导致绑定关系无法直接恢复时，系统需提示用户重新确认或修复映射关系。
- 统一筛选相关配置不应依赖源环境数据库主键或其他不可迁移内部标识。

### 9）YAML 导入导出一致性规则

- 仪表盘统一筛选定义、默认值及组件绑定关系需纳入 YAML 导入导出范围。
- 导出仪表盘时，统一筛选配置应作为画布配置的一部分完整导出，不得只导出组件布局而遗漏筛选语义。
- 导入仪表盘时，系统应恢复统一筛选定义、组件绑定关系及其与数据源的关联表达。
- YAML 中关于统一筛选及其关联关系的表达，需基于可迁移的业务标识或逻辑标识，而非数据库 ID。
- 当目标环境缺少统一筛选所依赖的数据源参数语义或数据源对象时，系统应在导入前给出明确提示。
- 导入后若存在无法自动恢复的统一筛选绑定关系，系统需提供明确的受影响对象清单。

### 10）权限与边界规则

- 统一筛选能力遵循现有仪表盘查看、编辑及数据源访问权限规则。
- 用户无权访问某组件的数据源时，不应通过统一筛选绕过权限获得结果。
- 统一筛选项的新增、编辑、删除权限语义应与仪表盘编辑权限保持一致。
- 统一筛选能力不要求改变现有拓扑、架构图画布的交互方式。
- 本期不要求支持跨仪表盘共享统一筛选状态。
- 本期不要求支持跨页面全局筛选、筛选方案市场、智能推荐筛选项或自动推断全部数据源映射关系。

## 适用范围

- 运营分析仪表盘统一筛选定义与使用
- 统一筛选与组件绑定关系配置
- 数据源参数分类与统一筛选映射表达
- 统一筛选随仪表盘保存、复制、复用
- 统一筛选相关 YAML 导入导出

## 非目标（本期不包含）

- 拓扑图、架构图画布统一筛选
- 图表点击联动、钻取联动、跨组件事件联动
- 复杂逻辑筛选编排、嵌套条件组合、筛选方案收藏
- 跨仪表盘共享筛选状态或跨页面全局筛选中心
- 动态脚本生成筛选项、自动学习全部数据源参数语义
- 任意深层 JSON 参数结构映射与复杂转换编排
- 基于统一筛选自动修复所有失效组件绑定关系

## 验收标准

- 仪表盘类型画布中可定义多个统一筛选项，且每个筛选项具备稳定标识、显示名称、值类型、展示控件和默认值。
- 仪表盘可配置为统一筛选项的字段，仅来自数据源中标记为“参数”的字段。
- 文档与实现对“参数”“固定参数”“组件私有参数”“公共筛选 key”的判定口径一致，不存在同一字段在不同模块中被重复定义为不同语义的情况。
- 统一筛选项至少支持单选下拉、多选下拉、关键字输入和时间范围四类输入形态。
- 用户修改统一筛选值后，当前画布内所有已绑定该筛选项的组件会按最新条件重新请求数据，未绑定组件不受影响。
- 当多个组件的数据源参数 key 相同，且这些参数属于可参与统一筛选的“参数”字段时，系统可基于该 key 创建公共筛选并同时作用于这些组件。
- 当多个组件的数据源参数 key 不相同，系统本期不会自动将其合并为同一个公共筛选项。
- 组件可显式配置统一筛选绑定关系。
- 数据源固定参数不会被统一筛选覆盖；统一筛选参数与组件私有参数同时存在时，系统行为稳定且结果可预期。
- 数据源参数可明确区分固定参数、统一筛选参数和组件私有参数。
- 对于允许被统一筛选接管的参数，系统可表达其业务语义，而不仅是技术参数名。
- 仪表盘保存后，统一筛选定义、默认值及组件绑定关系会随画布一并保存，并在再次打开时正确回显。
- 仪表盘复制、模板复用或跨环境迁移后，统一筛选定义与组件绑定关系可一并保留。
- YAML 导出时包含统一筛选定义、默认值和组件绑定关系；YAML 导入后可恢复相同语义。
- YAML 中统一筛选及其关联关系不依赖数据库 ID 表达；当目标环境缺少相关依赖时，系统可给出明确提示。
- 当数据源调整导致绑定失效时，系统可提示受影响组件和统一筛选项，避免用户在无感知情况下得到错误结果。
