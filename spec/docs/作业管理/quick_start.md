# 快速入门

本指南将引导您使用「快速执行」功能，以最快的方式完成在目标主机上的第一次批量脚本执行。

## 1. 前置条件

在开始之前，请确保您的目标主机满足以下任意一种纳管方式：
*   **有 Agent 模式**：目标主机已通过「节点管理」安装控制器并在系统中呈现「在线」状态。
*   **无 Agent (Ansible) 模式**：目标主机具备 SSH 访问条件，并在系统内正确关联了相匹配的凭证信息。

## 2. 分步操作指引

### 第一步：进入快速执行页面
登录控制台左侧侧边栏，点击导航「作业管理」>「快速执行」。

### 第二步：编写执行脚本
1. 在页面左侧的代码编辑区域上方选择您的「脚本语言」（例如：**Shell**）。
2. 在代码框中输入一段简单的状态探测指令，例如查看远端机器运行时间与内存：
    ```bash
    echo "=== System Uptime ==="
    uptime
    echo "=== Memory Usage ==="
    free -m
    ```

### 第三步：选择目标主机
1. 在页面右半部分的「目标选择」区域，点击对应的「＋添加」按钮。
2. 弹出拓扑结构域后，您可以通过现有的系统拓扑模块勾选需要纳入测试的业务机，或者直接通过“手动输入 IP”的方式填入目标主机。
3. 确认已选中的机器列表无误后，点击右下角确定闭合弹窗。

### 第四步：执行作业
1. 若无特权要求，可保持下方的“执行账户”和“超时时间”为系统默认值。
2. 点击页面最下方的主按钮「执行」来提交本次作业下发。

> **⚠️ 安全最佳实践：**
> 
> 在您点击执行前的一瞬间，该执行代码将经过「高危命令配置」层识别。如果您一时手快输入了类似的破坏性语句（如 `rm -rf /` ），这笔执行单据将被作业平台直接强制截阻拦截，全方位防止低级失误。

## 3. 结果验证与闭环

提交操作请求几乎是瞬间响应完成的，页面会自动跳转定位至该笔任务的诊断追踪页面层：

> **🖼️ 界面指引：**
> 
> *   **总运行成功率概览**：界面顶端的饼环或进度条会直观向您展示成功主机/失败主机的总体分档情况。
> *   **日志结果追踪**：点击下方主机表格中那台我们刚刚选定的目标节点，右侧或展开的弹窗将直接输出刚刚通过命令查询到的 `Uptime` 与 `Memory` 信息！如果命令退出回执存在异常（即 Exit Code 不为 0 ），直接在执行日志栏亦可以直观洞悉异常缘由。
