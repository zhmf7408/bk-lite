# 运营分析支持 YAML 导入导出 Solution Plan

日期：2026-04-07

## 1. 方案概览

### 1.1 目标

在不改变运营分析现有对象模型与主流程的前提下，新增一套统一的 YAML 导入导出能力，覆盖画布对象（仪表盘、拓扑、架构图）与配置对象（数据源、命名空间），满足跨环境迁移、模板复用、备份归档与批量下发场景。

### 1.2 方案原则

- 严格基于 Requirement 落地，不扩展目录树恢复、删除式同步、多格式导入导出等非目标能力。
- 轻量优先：复用现有运营分析前后端模块、权限体系与请求框架，避免引入重型编排。
- 一致性优先：页面能力与开放接口能力保持一致，返回语义一致。
- 可迁移优先：YAML 中对象身份与引用关系基于业务标识表达，禁止使用数据库 ID。

### 1.3 范围边界

#### In Scope

- 统一 YAML 规范（同一套导入与导出格式）。
- 画布对象导出（含组件配置）与实际依赖的数据源、命名空间、关系导出。
- 配置对象（数据源、命名空间）独立导入导出。
- 导入前校验、导入概览、冲突识别与处理（跳过/覆盖/另存为）。
- 开放接口：导出、导入提交、导入预检（导入提交同步返回结果）。
- 敏感字段导出脱敏与导入补全提示。

#### Out of Scope

- 目录结构导入导出与源目录树恢复。
- 运行时数据、历史结果、附件文件迁移。
- 删除式同步（按导入包清理目标环境未包含对象）。
- Excel/JSON/压缩包等其他格式导入导出。

### 1.4 总体实现思路

- 后端新增导入导出应用层服务，统一处理 YAML 序列化、校验、对象映射、冲突决策与结果汇总。
- 前端在运营分析入口补齐“导出/导入”操作流，提供预检结果与导入结果可视化反馈。
- 开放接口与页面共用同一后端能力层，避免双套逻辑导致结果不一致。
- 本期按同步执行模型落地：预检与导入均同步返回结果；导入提交接口直接返回本次执行结果明细。

## 2. 关键方案设计

### 2.1 统一 YAML 契约

- 顶层结构固定为 `meta` + 各对象章节。
- `meta` 至少包含：版本、导出时间、导出来源、对象统计。
- 对象章节按类型组织：`dashboards`、`topologies`、`architectures`、`datasources`、`namespaces`。
- 支持“部分章节导入导出”：仅保留相关章节即可，无需专用模板。
- 对象身份使用可迁移业务键（如对象名称+类型+必要归属信息），禁止主键/外键 ID。
- 引用关系使用逻辑标识（如数据源业务键、命名空间名称）表达。
- 画布对象的 `view_sets` 契约采用“与真实结构对齐”策略：仪表盘在 YAML 中直接使用数组，拓扑保持 `{ nodes, edges }`，架构图保持 `{ items, views }`，不为了表面统一额外包裹一层对象。

### 2.2 导出策略

- 画布导出：导出画布本体 + 组件配置 + 实际依赖的数据源/命名空间 + 关系映射。
- 配置对象导出：仅导出用户选择的配置对象及必要元数据，不夹带无关画布内容。
- 依赖收敛：仅导出被当前导出对象真实引用的依赖对象。
- 稳定性要求：同配置重复导出时，章节结构与字段顺序保持稳定，便于差异比对。
- 敏感字段处理：数据源敏感参数统一脱敏占位（如 `******` / `${SECRET}`），不导出明文。

### 2.3 导入与预检策略

- 导入前执行结构校验、字段校验、依赖完整性校验、非法 ID 引用校验。
- 支持导入概览：对象类型、数量、依赖关系、冲突清单。
- 画布导入必须显式指定目标目录。
- 配置对象导入无需目录信息，按章节直接处理。
- 导入仅执行新增/更新，不删除目标环境已有且未出现在 YAML 的对象。
- 关键依赖不可恢复时执行失败拦截，避免产生不可用半成品。

### 2.4 冲突处理策略

- 冲突识别范围：同名画布、同名数据源、同名命名空间等。
- 支持处理动作：跳过、覆盖、另存为。
- 另存为命名规则：追加后缀（如 `名称_copy`）。
- 导入前明确展示“将新增/将覆盖/将跳过/预期失败”对象明细。
- 导入结果返回实际处理明细与原因，便于审计与自动化编排。

### 2.5 开放接口策略

- 与页面能力一致，至少提供三类接口：
  - 导出接口
  - 导入提交接口
  - 导入预检接口
- 与页面能力一致，开放接口仅提供三类能力，不提供结果查询接口。
- 接口返回结构必须可区分对象级状态：新增、覆盖、跳过、失败。
- 接口认证鉴权沿用现有平台机制，权限语义与页面一致。
- 接口导入画布同样强制目标目录参数。

## 3. 分阶段落地计划

### 阶段 1：契约与后端基础能力（M1）

#### 目标

完成统一 YAML 契约、导出能力与预检能力的后端闭环。

#### 交付

- 定义统一 YAML 字段与章节规范。
- 新增导出能力（画布对象、配置对象、依赖收敛）。
- 新增预检能力（结构/字段/依赖/非法 ID 校验）。
- 完成敏感字段脱敏导出策略。

#### 验收

- 可导出单个画布或配置对象并得到统一 YAML。
- 导出 YAML 不包含数据库 ID。
- 预检可返回对象清单、冲突信息、失败原因。

### 阶段 2：导入执行与冲突处理（M2）

#### 目标

完成同步导入执行、冲突处理与关联恢复。

#### 交付

- 导入提交能力（同步执行）。
- 冲突策略实现（跳过/覆盖/另存为）。
- 画布导入目标目录约束。
- 对象关联恢复与失败拦截。

#### 验收

- 导入后对象可正常打开、展示、编辑。
- 可按策略处理冲突并返回对象级结果。
- 导入过程不删除目标环境原有未命中对象。

### 阶段 3：前端交互与开放接口收口（M3）

#### 目标

打通页面导入导出体验，并保证页面与接口能力一致。

#### 交付

- 运营分析页面导入导出入口与流程。
- 预检结果展示、导入概览展示、导入结果摘要展示。
- 开放接口三类能力联调完成。
- 导入提交接口直接返回对象级状态明细。

#### 验收

- 用户可在页面完成导出、预检、导入、结果查看。
- 外部系统可通过接口完成同等流程。
- 页面与接口在同一 YAML 输入下得到一致结果。

## 4. 验收清单（可执行）

### 4.1 功能验收

- 支持画布对象导出与配置对象导出。
- 支持 YAML 上传预检、导入提交（同步返回结果）。
- 画布导入必须指定目标目录。
- 导入完成后对象可访问、可展示、可编辑。

### 4.2 规则验收

- 导入导出使用同一 YAML 规则。
- YAML 不包含数据库 ID，引用关系基于业务标识。
- 导出仅包含实际依赖，不夹带无关对象。
- 导入仅新增/更新，不执行删除式同步。

### 4.3 安全与权限验收

- 敏感字段导出为脱敏占位，不明文泄露。
- 导入导出接口遵循平台认证鉴权。
- 页面与接口权限语义一致，未授权不可执行。

### 4.4 结果反馈验收

- 预检可清晰返回风险与问题清单。
- 导入结果可区分成功、失败、跳过、覆盖数量与对象明细。
- 对无法自动恢复依赖项给出明确后续处理提示。

## 5. 依赖与协同

### 5.1 后端依赖

- 运营分析对象模型（目录、画布、数据源、命名空间）现有能力复用。
- 组织与权限过滤能力复用。
- 统一错误码与返回结构规范复用。

### 5.2 前后端协同要点

- 统一 YAML 章节名、字段名、枚举值定义。
- 统一冲突策略参数协议。
- 统一结果明细结构（对象标识、处理动作、状态、原因）。

### 5.3 外部集成协同

- 明确开放接口调用顺序建议：预检 -> 导入提交。
- 对接方需提供目标目录、冲突策略、必要映射参数。

## 6. 风险与应对

### 风险 1：现有配置中引用了数据库 ID

- 影响：历史对象导出后可能无法按新规则直接迁移。
- 应对：导出时统一转换为业务标识；预检阶段严格拦截非法 ID 引用并提示修复。

### 风险 2：冲突处理后引用关系断裂

- 影响：对象导入成功但运行时不可用。
- 应对：冲突决策与关系恢复在同一执行单元处理；关键依赖无法恢复时整体失败拦截。

### 风险 3：敏感字段脱敏后导入不可直接可用

- 影响：用户误判导入后即刻可用。
- 应对：导入结果显式提示需补充认证信息字段，并列出受影响对象。

### 风险 4：同步模式下大包导入耗时高

- 影响：请求超时与体验不稳定。
- 应对：限制单次导入包大小与对象数量上限，超限返回明确错误；后续按需要评估异步化。

## 7. 回滚与降级策略

- 发布异常时可关闭前端导入导出入口，保留现有运营分析核心能力。
- 后端接口异常时，按接口级错误返回，不影响现有 CRUD 与取数能力。
- 导入执行失败按事务回滚本次写入，避免部分对象落库。
- 对已导入成功对象不做自动删除回滚，按结果明细由人工选择补救策略。

## 8. 遗漏项检查（本次已执行）

- 范围边界：已明确 In Scope / Out of Scope，并与 Requirement 保持一致。
- 依赖方：已覆盖前端、后端、开放接口调用方三类协同角色。
- 验收口径：已给出可执行、可回归的功能/规则/安全/反馈验收项。
- 风险与回滚：已定义主要失败路径、降级策略、回滚策略。

## 9. 需人工判断/确认

- 需人工判断：发布策略（灰度范围、放量节奏、回滚阈值）选择。
- 需人工确认：同步执行模式下的请求超时阈值与导入包大小上限。
- 需人工确认：冲突“覆盖”的字段粒度是全量覆盖还是仅覆盖 YAML 声明字段。
