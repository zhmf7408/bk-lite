概述

Docker 是最流行的容器运行时，容器日志是排查应用问题的重要数据来源。本集成使用 Vector 的 docker_logs source 采集 Docker 容器的标准输出（stdout）和标准错误（stderr）日志。

Vector 通过 Docker API 自动发现运行中的容器，并实时采集其日志输出。支持按容器名称、镜像、标签等条件过滤，以及多行日志合并、JSON 解析等高级处理功能。

日志字段说明

字段名 类型 说明
message string 容器输出的日志内容
stream string 输出流类型：stdout 或 stderr
container_name string 容器名称
container_id string 容器 ID（完整 64 字符）
image string 容器使用的镜像名称和标签
labels object 容器的 Docker 标签
host string Docker 宿主机名称
timestamp datetime 日志产生时间

最佳实践

  • 使用标签过滤：通过 Docker 标签（如 logging=true）精确控制采集范围，避免采集无关容器
  • 排除采集器容器：确保排除 Vector 自身的容器，避免日志循环
  • 配置日志轮转：在 Docker daemon 配置日志大小限制，防止日志文件过大
  • 应用输出 JSON：建议应用以 JSON 格式输出日志，便于结构化解析和检索
  • 避免敏感信息：确保容器日志不包含密码、密钥等敏感信息
  • 设置资源限制：为 Vector 容器设置 CPU/内存限制，防止影响宿主机性能
