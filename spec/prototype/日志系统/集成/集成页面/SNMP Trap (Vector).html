SNMP Trap (Vector)

接收 SNMP Trap 消息，获取网络设备、服务器、存储等发送的异常事件通知

⚙️ 配置
📖 简介
配置
请选择已托管 snmptrapd 程序的节点
接入指引
📟
请先选择节点以查看接入指引
1
配置 Trap 目标地址
在设备上配置以下目标地址发送 Trap 消息
目标 IP： -
目标端口： 162
2
配置 MIB 文件
从设备官方获取 MIB 文件，放至节点以下目录
目录路径： /usr/share/mibs

概述

SNMP Trap 接收使用 Vector 作为采集器，提供高性能的 SNMP Trap 服务端能力。可以接收来自网络设备、服务器硬件（iDRAC/iLO）、存储设备、UPS、环境监控等发送的异常事件通知。

功能特性

🔔

实时告警

即时接收设备主动推送的异常事件，无需轮询

📋

多版本支持

支持 SNMPv1、v2c、v3 协议版本

🔍

OID 解析

自动解析 OID 为可读名称，支持自定义 MIB

⚡

高吞吐量

单节点支持每秒数万条 Trap 消息接收

解析字段

SNMP Trap 消息将被解析为以下结构化字段：

字段名 类型 说明
@timestamp date Trap 接收时间戳
snmp.source_ip ip 发送设备 IP 地址
snmp.version keyword SNMP 版本 (v1, v2c, v3)
snmp.community keyword Community 字符串 (v1/v2c)
snmp.trap_oid keyword Trap OID 标识符
snmp.trap_name keyword Trap 名称（通过 MIB 解析）
snmp.enterprise keyword Enterprise OID (v1)
snmp.varbinds object 变量绑定列表（OID-值对）

配置发送端

配置网络设备将 SNMP Trap 发送到 Vector：

⚠️
Cisco 设备配置示例：
snmp-server host 172.24.0.5 traps version 2c public
snmp-server enable traps

华为设备配置示例：
snmp-agent target-host trap address udp-domain 172.24.0.5 params securityname public v2c
