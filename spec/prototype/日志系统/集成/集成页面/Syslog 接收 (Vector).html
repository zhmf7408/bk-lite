Syslog 接收 (Vector)

接收 Syslog 协议日志，支持 RFC3164 和 RFC5424 格式，TCP/UDP 传输

⚙️ 配置
📖 简介
配置
请选择已托管 Vector 采集器的容器节点
接入指引
📋
请先选择节点以查看接入指引
在设备上配置 Syslog 目标地址
请在您的设备或应用程序中配置以下Syslog服务器地址，以便将日志消息发送到此接收器
目标 IP： -
目标端口： 514(UDP) / 1514(TCP)

概述

Syslog 接收使用 Vector 作为采集器，提供高性能的 Syslog 服务端能力。可以接收来自网络设备、Linux 服务器、容器等各类基础设施发送的 Syslog 日志。

功能特性

🔄

双协议支持

同时支持 UDP 和 TCP 传输，满足不同场景需求

📋

格式兼容

自动识别 RFC3164 (BSD) 和 RFC5424 格式

🔒

TLS 加密

TCP 模式支持 TLS 加密传输，保障数据安全

⚡

高吞吐量

单节点支持每秒数十万条日志接收

解析字段

Syslog 日志将被解析为以下结构化字段：

字段名 类型 说明
@timestamp date 日志时间戳
syslog.facility keyword 设施类型 (kern, user, mail, daemon 等)
syslog.severity keyword 严重级别 (emerg, alert, crit, err, warning, notice, info, debug)
syslog.hostname keyword 发送主机名
syslog.appname keyword 应用名称 / 进程名
syslog.procid keyword 进程 ID
syslog.msgid keyword 消息 ID (RFC5424)
message text 日志消息内容

配置发送端

配置 Linux 系统将 Syslog 发送到 Vector：

⚠️
rsyslog 配置示例：
编辑 /etc/rsyslog.conf，添加以下行：
*.* @172.24.0.5:514 (UDP) 或 *.* @@172.24.0.5:1514 (TCP)
然后重启 rsyslog：systemctl restart rsyslog
