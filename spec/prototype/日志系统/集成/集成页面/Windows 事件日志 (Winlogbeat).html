概述

Windows 事件日志是 Windows 操作系统的核心日志系统，记录了系统运行、应用程序和安全相关的所有事件。本集成使用 Winlogbeat 采集 Windows 事件日志，Winlogbeat 是 Elastic 官方提供的专用 Windows 事件日志采集器。

Windows 事件日志对于安全监控、故障排查和合规审计至关重要。通过采集 Security、System、Application 等日志通道，可以全面掌握 Windows 系统的运行状态和安全态势。

最佳实践

  • 启用审计策略：确保 Windows 审计策略已正确配置，否则相关事件不会被记录
  • 关注登录事件：4624（成功）和 4625（失败）是最重要的安全事件，务必采集
  • 监控特权操作：4672（特权登录）和 4688（进程创建）有助于检测提权攻击
  • 安装 Sysmon：建议安装 Microsoft Sysmon 增强安全监控能力
  • 启用 PowerShell 日志：PowerShell 是攻击者常用工具，建议启用脚本块日志
  • 合理过滤：使用事件 ID 过滤减少噪音，但要确保关键事件不被遗漏
