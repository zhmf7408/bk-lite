请求体填写说明
请求体格式
<metric_id>,<tag_set> <field_set> <timestamp>
一行表示一个指标点。批量上报时，在同一请求体中按换行拼接多条记录即可，每一行都遵循上面的格式；<timestamp> 可选，不传时默认使用服务端接收时间。
标识 标识含义 是否必填 说明 示例写法
<metric_id> 指标 ID 是 每一行对应一个指标，填写当前上报指标的唯一标识 mysql_connections_active
<tag_set> 维度标签集合 是 多个标签使用逗号分隔，建议至少包含 cloud_area、organization、host cloud_area=华东-生产,organization=Default,host=Default-mysql-demo,env=prod
<field_set> 指标字段集合 是 当前格式下建议统一写成 value=<metric_value>；整数用 i 结尾，浮点数直接写 value=150i
<timestamp> 采集时间戳 否 位于每行末尾，使用纳秒级 Unix 时间戳；如未传入，默认使用服务端接收时间 1710576000000000000
