## 2. 核心数据结构 / Schema 定义

```python
from dataclasses import dataclass
from typing import Literal

@dataclass(slots=True)
class TableDefaultColumn:
  """表格列定义（数据源默认列与组件列使用同一结构）。"""
    key: str                  # 字段唯一键（与数据 key 对齐）
    title: str                # 列标题
    visible: bool             # 默认是否展示
    order: int                # 展示顺序，0 开始递增
    width: int | None = None  # 列宽，可空


@dataclass(slots=True)
class DataSourceTableField:
    """数据源配置页维护的“接口返回字段定义”（同时作为默认列）。"""
    key: str
    title: str
    value_type: Literal["string", "number", "boolean", "datetime"]
    description: str | None = None


@dataclass(slots=True)
class DataSourceTableSchemaConfig:
    """表格返回字段定义，可为空；非空时作为默认列来源。"""
    fields: list[DataSourceTableField]


@dataclass(slots=True)
class TableFilterField:
  """table 组件配置页可维护的筛选字段定义。"""
  key: str
  label: str
  input_type: Literal["keyword", "time_range", "select"]
  options: list[str] | None = None


@dataclass(slots=True)
class TableWidgetConfig:
  """table 组件配置（组件内可保存并复用）。"""
  columns: list[TableDefaultColumn]
  filter_fields: list[TableFilterField]
```

**DataSourceAPIModel 新增字段定义（Django Model）**

```python
# server/apps/operation_analysis/models/datasource_models.py
source_table_schema_config = models.JSONField(
    default=dict,
    blank=True,
    help_text="接口返回字段定义（table 可选，亦作为默认列来源）"
)
```

字段约束：

- `source_table_schema_config.fields` 可为空。
- 当 `source_table_schema_config.fields` 非空时，`fields[].key` 必须唯一且非空。
- 命名空间标识口径固定为 `namespace.name`（展示与筛选同口径）。

**数据源配置页（operateModal）规则**

- 允许用户在数据源配置页维护“接口返回字段定义”（`source_table_schema_config.fields`）。
- 该配置为可选项：可以不填。
- 该配置非空时，table 组件将其作为默认展示列来源。
- 该配置为空时，table 组件必须在仪表盘组件配置中单独维护展示列。

## 4. 核心逻辑伪代码 (Step-by-Step Logic)

### 4.1 列配置决策（最复杂逻辑）

```text
1) 初始化配置
  - 读取数据源 source_table_schema_config.fields，映射为默认列 defaultColumns。
  - 读取组件 config.columns，作为组件列 componentColumns。
  - 若 defaultColumns 与 componentColumns 均为空：拉取第 1 页数据，用首条记录 key 生成推断列 inferredColumns。

2) 列集归一化
  - 以 defaultColumns 为基线，与 componentColumns 按 key 合并。
  - 合并规则：同 key 时组件列覆盖默认列（title/visible/order/width）。
  - 若某 key 仅存在于组件列，追加到结果列。
  - 若默认列与组件列均为空，使用 inferredColumns 作为结果列。
  - 对结果列按 order 排序，缺失 order 的列按追加顺序放末尾。
  - 生成生效列 effectiveColumns。

  2.1) 筛选字段配置归一化
    - 从组件配置读取 filterFields。
    - 未配置时默认生成：keyword、time_range、namespace（多命名空间时）。
    - 已配置时校验 key 唯一，非法配置项丢弃并提示。

  2.2) 组件配置页列编辑能力
    - 允许在配置页新增一行列配置，行内输入 key 与 displayName(title)。
    - 允许删除任意列配置行。
    - 允许拖拽列配置行调整顺序，并同步写回 order。
    - 同 key 行冲突时阻断保存并提示。

3) 持久化与返回
   - 将 effectiveColumns 回写组件配置（仅当前 widget）。
  - 前端按 effectiveColumns 渲染表格。
```

### 4.2 查询与分页执行流

```text
1) 前端触发
  - 首次加载与筛选条件变化均通过现有 fetchWidgetData -> getSourceDataByApiId 调用 get_source_data。
  - 前端仅透传现有参数并刷新组件数据，不执行本地分页。

2) 后端执行
   - 保持 get_source_data 原有逻辑，不新增 table 分支处理。
  - get_source_data 仅透传参数与结果，不承担分页逻辑。
  - 实际分页由 get_source_data 下游转发的 NATS 接口完成。

3) 前端展示
  - 直接渲染 get_source_data 返回的分页结果，不做二次分页。
  - 组件内错误态与空态处理保持与现有组件一致。
```

### 4.3 数据源配置与组件配置协同

```text
1) 数据源配置页
   - 用户可选填写 source_table_schema_config.fields（接口返回字段定义）。

2) 仪表盘 table 组件
   - 若数据源 fields 非空：作为默认列基线。
  - 首次打开组件配置页且组件无列配置时，自动将数据源 fields 填充为默认展示列。
   - 若组件额外配置了展示列：与默认列按 key 合并展示。
   - 若数据源 fields 为空：用户在 table 组件配置页手动配置展示列。
  - 组件配置页允许维护筛选字段（至少支持 keyword、time_range；多命名空间时支持 namespace）。
  - 组件配置页允许维护展示列：可新增 key/display 行并拖拽调整顺序。

3) 最终渲染
   - 生效列 = merge(defaultColumns, componentColumns)。
  - 生效筛选项 = normalize(componentFilterFields)。
```

### 4.4 异常状态隔离

```text
1) 单组件查询失败 -> 仅该组件显示失败态，不影响其他组件。
2) 无权限 -> 该组件显示无权限态，不触发页面级错误。
3) 空数据 -> 显示空态并保留列配置入口。
```

## 技术决策固化

- 接口策略：不新增 table 专用接口，复用现有 `get_source_data` 且保持原逻辑不变。
- 分页口径：分页由下游 NATS 接口负责，get_source_data 仅透传，前端只渲染返回结果。
- 默认列归属：默认列即数据源返回字段定义，存储在 `DataSourceAPIModel.source_table_schema_config`。
- 返回字段定义：允许为空；为空时由组件侧独立配置展示列。
- 合并规则：组件配置列与数据源返回字段定义按 key 合并，组件同 key 配置优先。
- 命名空间口径：统一使用 `namespace.name`。
